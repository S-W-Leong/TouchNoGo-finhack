# .impeccable — TNG RiskOps Agent Design Context

## Design Context

### Users
Finance and fraud analysts at TNG making pre-payout decisions under time pressure. Power users who live in this tool all day — scanning for signals, building a case, taking an action. They need maximum information density, zero ambiguity about system state, and fast visual scanning. The secondary audience is hackathon judges getting a 2-minute look who need to immediately read what they're seeing.

### Brand Personality
TNG (Touch 'n Go) fintech operations. Three words: **precise, authoritative, operational**.

Emotional goal: confidence in the decision, trust in the evidence, urgency without panic. The UI should feel like a professional instrument, not a marketing page or an AI demo.

### Aesthetic Direction
**Primary reference: Bloomberg Terminal** — extreme information density, high contrast, color as signal only, monospace data everywhere. This is a workspace, not a dashboard.

**TNG brand colors as the organizing palette** — blue for navigation and interaction, orange for high-priority actions and alerts. The orange naturally doubles as the brand identity and the "act now" risk signal. That double duty is elegant and intentional.

**Dark mode by default** — deep navy background, not pure black. Bright text on dark surfaces. No light mode needed for the hackathon build.

**Anti-references — do not build these:**
- Purple/blue gradient backgrounds or glowing card borders
- Frosted glass / backdrop-blur surfaces
- Oversized rounded cards (no radius > 8px on data surfaces)
- Animated gradient hero sections or "AI-thinking" particle effects
- Neumorphism or soft-shadow depth tricks
- Any visual pattern that says "AI startup SaaS" instead of "financial infrastructure"

### Design Principles
1. **Evidence density over decoration** — every pixel earns its place; whitespace is for separation, not aesthetics; if a piece of chrome isn't load-bearing, remove it
2. **Color is signal, never style** — TNG orange = act now / high-risk; TNG blue = navigate / interact; red = blocked / frozen; green = clear / allow; never use color decoratively
3. **Monospace for all data** — IDs, scores, amounts, timestamps, hashes, boolean flags: always IBM Plex Mono; this is non-negotiable and is the single strongest visual differentiator from generic SaaS
4. **Contrast over depth** — no blur, no glass, no shadows; borders and luminance do the separation work; dark surfaces with bright foregrounds
5. **Hierarchy through luminance** — primary content near-white `#E8ECF0`, secondary medium grey `#8A9BB0`, tertiary dim grey `#55687A`; reserve color for status only

## Design System

### Color Tokens (dark mode baseline)

```css
:root {
  /* Backgrounds */
  --bg:               #080d14;   /* page background — deep navy near-black */
  --surface:          #0f1825;   /* panel background */
  --surface-raised:   #162033;   /* elevated panel, nested cards */
  --surface-hover:    #1c2a3f;   /* row hover, selected state */
  --surface-input:    #111b2a;   /* input fields */

  /* Borders */
  --line:             rgba(255,255,255,0.08);   /* default divider */
  --line-strong:      rgba(255,255,255,0.15);   /* panel edge, focused input */

  /* Text */
  --text:             #e8ecf0;   /* primary — readable, not stark white */
  --text-dim:         #8a9bb0;   /* secondary labels, supporting text */
  --text-muted:       #55687a;   /* eyebrows, placeholders, tertiary */

  /* TNG Brand */
  --tng-blue:         #0066cc;   /* interactive: links, active nav, focus rings */
  --tng-blue-dim:     rgba(0,102,204,0.18);
  --tng-orange:       #ff6600;   /* high-priority: CTAs, HIGH risk, primary action */
  --tng-orange-dim:   rgba(255,102,0,0.15);

  /* Semantic Status */
  --status-critical:  #e03535;   /* CRITICAL / FREEZE_ACCOUNT / BLOCKED */
  --status-high:      #ff6600;   /* HIGH — reuses TNG orange intentionally */
  --status-warn:      #f0a500;   /* STEP_UP_VERIFICATION / PENDING */
  --status-ok:        #1db86a;   /* ALLOW / CLEAR / SUCCESS */
  --status-escalate:  #a78bfa;   /* ESCALATE / ESCALATED */
  --status-info:      #38bdf8;   /* MEDIUM / informational */
  --status-neutral:   #55687a;   /* muted state, no action */

  /* Radius — tight, Bloomberg-adjacent */
  --r-xs:   2px;   /* data cells, micro-chips */
  --r-sm:   4px;   /* status badges, tags */
  --r-md:   6px;   /* panels, buttons, inputs */
  --r-lg:   8px;   /* sheets, modals, primary CTAs */

  /* Typography */
  --font-ui:   var(--font-manrope), sans-serif;
  --font-data: var(--font-plex-mono), monospace;
}
```

### Typography Rules

| Element | Font | Size | Weight | Treatment |
|---|---|---|---|---|
| Page title / product name | Instrument Sans | 0.9rem | 700 | normal |
| Section heading | Instrument Sans | 0.875rem | 650 | normal |
| Body / list items | Instrument Sans | 0.8125rem | 400 | normal |
| Eyebrow label | IBM Plex Mono | 0.65rem | 500 | UPPERCASE 0.1em tracking |
| Metric / score | IBM Plex Mono | clamp(1.4rem, 2vw, 2rem) | 600 | tabular-nums |
| Data cell (ID, amount, timestamp) | IBM Plex Mono | 0.75rem | 400 | tabular-nums |
| Status badge | IBM Plex Mono | 0.65rem | 600 | UPPERCASE |
| Button label | Instrument Sans | 0.8125rem | 650 | normal |

**Rule**: If it's a number, a code, an ID, or a timestamp — it's monospace. Always.

### Component Patterns

#### Panel / Card
```
background: var(--surface)
border: 1px solid var(--line-strong)
border-radius: var(--r-md)
no box-shadow
```
Elevated panels (e.g. a detail card nested inside a panel) use `--surface-raised`.

#### Data Row (dense list)
```
background: transparent
border-bottom: 1px solid var(--line)
padding: 0.6rem 0.75rem
hover: background var(--surface-hover)
```
No row borders on sides. Horizontal rules only.

#### Status Badge
```
font: IBM Plex Mono 0.65rem 600 UPPERCASE
border-radius: var(--r-xs)   ← NOT pill-shaped
padding: 2px 6px
color + background from semantic status vars
```
Bloomberg Terminal uses rectangular status indicators. Pills are a consumer SaaS pattern.

#### Button — Primary Action (TNG Orange)
```
background: var(--tng-orange)
color: #000   ← black text on orange for contrast
border-radius: var(--r-md)
padding: 0.5rem 1rem
font: Manrope 0.8125rem 650
```

#### Button — Secondary
```
background: var(--surface-raised)
border: 1px solid var(--line-strong)
color: var(--text)
border-radius: var(--r-md)
```

#### Button — Danger (Freeze / Override)
```
background: rgba(224,53,53,0.12)
border: 1px solid rgba(224,53,53,0.25)
color: var(--status-critical)
border-radius: var(--r-md)
```

#### Input / Search
```
background: var(--surface-input)
border: 1px solid var(--line-strong)
focus: border-color var(--tng-blue), outline none
border-radius: var(--r-md)
font: Manrope 0.8125rem
color: var(--text)
placeholder: var(--text-muted)
```

#### Score / Metric Display
```
font: IBM Plex Mono, large
color: --text (near-white) for neutral
color: --tng-orange for high-risk scores
color: --status-critical for critical/freeze scores
background: --surface-raised
border-left: 3px solid [semantic color]   ← Bloomberg uses colored left-borders for state
```

### Nav / Shell
- Background: `--bg` (page color, no separate nav bar background)
- Active nav item: `border-bottom: 2px solid var(--tng-orange)` — no filled background pill
- Product name: `--tng-blue` for the TNG mark, white for "RiskOps Agent"
- Breadcrumb/context: IBM Plex Mono, `--text-muted`

### What to replace from the current design

| Current | Replace with |
|---|---|
| `--bg: #e9eef5` (light) | `--bg: #080d14` |
| `--surface: rgba(255,255,255,0.92)` | `--surface: #0f1825` |
| `backdrop-filter: blur(10px)` | Remove entirely |
| `--radius-xl: 18px` / `--radius-lg: 14px` | `--r-md: 6px` / `--r-lg: 8px` |
| `--accent: #0f766e` (teal) | `--tng-blue: #0066cc` + `--tng-orange: #ff6600` |
| Pill-shaped status badges (`border-radius: 999px`) | `border-radius: var(--r-xs)` (2–4px) |
| `box-shadow: 0 8px 24px ...` | Remove — borders only |
| Nav active state: filled dark pill | `border-bottom: 2px solid var(--tng-orange)` |
| Radial gradient background | Solid `--bg` only |

### Spacing Scale
Use a 4px base: 4 / 8 / 12 / 16 / 20 / 24 / 32. No 3px or 7px gaps — they read as accident rather than intent.

### Do Not
- Add any glow, neon, or light-emission effect
- Use gradients on interactive elements
- Round data table cells beyond `--r-xs`
- Use color to add "visual interest" — only for semantic state
- Use more than 3 colors in any single view (bg, text, one status signal)
- Add icons that are purely decorative — every icon must label an action or state
